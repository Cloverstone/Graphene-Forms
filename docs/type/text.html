Text field Options

All options are optional and can be left empty

Name types default description
namestringlabel exists -> field.label.toLowerCase().split(' ').join('_')
no label -> name is generated i.e. 'g1'		This value is is the variable name used
labelstring | boolfield.nameThis is the label displayed for the field, if you want to hide the label then set this to false

The string can also contain mustache i.e.
'My choice {{value}}'
valuestring | function''Function will run once at instantiation
helpstring''This text is used to provide context or help in filling out the field
placeholderstring''This text will be visible in the text box until you focus on the field
limitinteger0 (no limit)This is used to set the maxLength property and limits the number of characters that can be entered.
targetselector | nodeIf the target is not set then the field is appended to the parent element which is usually the main form but may be a fieldset
columnsinteger12The form is divided into a grid 12 columns wide, this is setting how many of those columns to display this element at, using this you can make split column forms
offsetinteger0This is the offset to the left to place this field
arrayboolean | objectfalseif set to true or and object then this field will have buttons for adding and removing instances of the item. Additionally the value will become an array of all instances of this field. Further information on this can be found here.
displayconditiontrueThis option controls whether this field should be visible to the user, allowing for fields to only be used if certain conditions are met
enableconditiontrueSimilar to display but this option controls whether the field is enabled for interaction
parsecondition'display'This option determins if the field value will be in the data result from myform.get()
validatearray[]This is a set of validations to perform on this field when validate() is called
requiredconditionfalseThis is a special case for validation that is calculated live and updates the label to indicate the required status. Validation of this happens when validate() is called

examples


Basic

Any field needs either a label or a name to be defined.

If only a label is defined then the name is derived from the label by replacing all spaces with underscores and lowercased, in the following example the name would be "label_example".

If just the name is provided then the name is also used for the label.

new gform(fields: [
  {label: 'Label Example'},	
  {name: 'Name Example'}
]);
